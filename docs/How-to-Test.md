# How to Test

## Overview
This document explains how to execute the test suites generated by the AI assistants across the Web, API, and Mobile domains.

## Running Playwright Web Tests

1. Navigate to the project directory:
   ```bash
   cd sample-project/playwright-web
   ```
2. Run all tests in headless mode (default):
   ```bash
   npm test
   ```
3. Run tests in headed mode (visible browser):
   ```bash
   npm run test:headed
   # or
   npx playwright test --headed
   ```
4. Run tests for a specific browser:
   ```bash
   npm run test:chromium
   ```
5. View the HTML report:
   ```bash
   npm run test:report
   # or
   npx playwright show-report
   ```

## Running Playwright API Tests

1. Navigate to the API project directory:
   ```bash
   cd sample-project/playwright-api
   ```
2. Execute the API tests:
   ```bash
   npm test
   ```
3. Ensure your `.env` file is properly configured with `API_BASE_URL`.

## Running Robot Framework Tests

1. Start the Appium server in a separate terminal:
   ```bash
   appium
   ```
2. Navigate to the Robot Framework project:
   ```bash
   cd sample-project/robot-framework
   ```
3. Run the test suite:
   ```bash
   robot -d reports tests/
   ```
4. Run tests with specific tags:
   ```bash
   robot -d reports -i smoke tests/
   ```
5. Override variables for Android execution:
   ```bash
   robot -d reports -V variables/android_vars.robot tests/
   ```

## Test Reports
* **Playwright**: Generates a self-contained HTML report in the `playwright-report/` directory.
* **Robot Framework**: Generates three files in the `reports/` directory:
  * `output.xml` (raw data)
  * `log.html` (detailed execution steps)
  * `report.html` (high-level summary)

## CI/CD Integration
To run these tests in GitHub Actions, ensure your workflow installs the necessary dependencies (`npx playwright install --with-deps` and sets up Python/Java environments). Use the `playwright-report` and `reports` directories as workflow artifacts.
